O que são components ?

O Component é simplesmente a Controller da View. Sim, a view é o HTML mesmo. É no component que nós cuidamos do que é necessário para trazer funcionalidade para as views, usando o Typescript para escrever o código. Por exemplo, é no component que nós usamos uma service para tazer valores e então passamos esses valores para alguma variável, para então poder exibí-la na view.

É convencionado no Angular nomear esses arquivos mais ou menos assim => exemplo.component.ts. Vale lembrar que estou me referindo ao que colocamos no arquivo .component. Dentro do component nós usamos um decorator chamado @Component para então declarar nossa View, o estilo(CSS,SASS e etc) e um seletor(nome do componente, que poderá ser chamado em outras views).

Pré-requisitos

Para criar um componente, verifique se está seguindo estes pré-requisitos

  1. Instale o Angular CLI com o seguinte comando: npm install g @angular/cli
  2. Veja se ja esta com o projeto criado. Se ainda nao possui um projeto, crie um usando o seguinte comando no prompt de comando: ng new nome-do-projeto, onde nome-do-projeto é o nome da sua aplicação em Angular
  3. Digite o comando: ng serve --open. Seu navegador irá abrir com a pagina do Angular informando que seu projeto está funcionando.

Criando um component

A melhor maneira de criar um component é com a Angular CLI. Você também pode criar um component manualmente.

Criando um component usando Angular CLI

Para criar um component usando o Angular CLI:

  1. Na janela do terminal, navegue até o diretório que contenha sua aplicação.
  2. Coloque o seguinte comando: ng generate component nome-do-componente, onde nome-do-componente é o nome do novo component.

Por padrão, este comando cria o seguinte:

  • O nome da pasta
  • O arquivo component, nome-do-componente.component.ts
  • O arquivo HTML, nome-do-componente.component.html
  • O arquivo CSS, nome-do-componente.component.css
  • O arquivo de teste, nome-do-componente.component.spec.ts

Onde nome-do-componente é o nome do seu component.
