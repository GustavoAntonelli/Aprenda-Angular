O que são modulos ?

Os Modules diferentemente dos components não controlam nenhuma view(html). Um Module se constitui de um ou mais components, ou seja, é lá que você registra seus components. Uma aplicação Angular tem de ter no mínimo um Module que contenha mínimo um Component.

Além de registrar components, outras responsabilidades do Module são declarar qual ou quais components podem ser usados por components que são de outros Modules, quais services serão injetados e qual componente será o componente inicial. Os Modules gerenciam os components e é com eles que nós podemos modularizar nossa aplicação.

Pré-requisitos

Para criar um componente, verifique se está seguindo estes pré-requisitos

  1. Instale o Angular CLI com o seguinte comando: npm install g @angular/cli
  2. Veja se ja esta com o projeto criado. Se ainda nao possui um projeto, crie um usando o seguinte comando no prompt de comando: ng new nome-do-projeto, onde nome-do-projeto é o nome da sua aplicação em Angular
  3. Digite o comando: ng serve --open. Seu navegador irá abrir com a pagina do Angular informando que seu projeto está funcionando.

Criando um modulo

A melhor maneira de criar um modulo é com a Angular CLI. Você também pode criar um modulo manualmente.

Criando um modulo usando Angular CLI

Para criar um modulo usando o Angular CLI:

  1. Na janela do terminal, navegue até o diretório que contenha sua aplicação.
  2. Coloque o seguinte comando: ng generate module nome-do-modulo, onde nome-do-modulo é o nome do novo modulo.

Por padrão, este comando cria o seguinte:

  • O nome da pasta
  • O arquivo module, nome-do-module.module.ts

Onde nome-do-componente é o nome do seu module.
