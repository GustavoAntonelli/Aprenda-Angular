O que são services ?

Nossos componentes precisam acessar os dados. Você pode escrever código de acesso a dados em cada componente, mas isso é muito ineficiente e quebra a regra da responsabilidade única. O componente deve se concentrar na apresentação de dados ao usuário. A tarefa de obter dados do servidor back-end deve ser delegada a alguma outra classe. Chamamos essa classe de classe de serviço. Porque fornece o serviço de fornecer dados a todos os componentes que deles necessitem.

Os componentes não devem buscar ou salvar dados diretamente e certamente não devem apresentar dados falsos intencionalmente. Eles devem se concentrar em apresentar dados e delegar acesso a dados a um serviço.

Em síntese, Service um pedaço de código reutilizável com um propósito específico. Um código que você usará em muitos componentes em seu aplicativo.

Um comparação seria o seguinte: o MODULO é a casa, a lampada o COMPONENTE e o que faz ela acender é o SERVICE.

Pré-requisitos

Para criar um service, verifique se está seguindo estes pré-requisitos

  1. Instale o Angular CLI com o seguinte comando: npm install g @angular/cli
  2. Veja se ja esta com o projeto criado. Se ainda nao possui um projeto, crie um usando o seguinte comando no prompt de comando: ng new nome-do-projeto, onde nome-do-projeto é o nome da sua aplicação em Angular
  3. Digite o comando: ng serve --open. Seu navegador irá abrir com a pagina do Angular informando que seu projeto está funcionando.

Criando um servico

A melhor maneira de criar um service é com a Angular CLI. Você também pode criar um service manualmente.

Criando um service usando Angular CLI

Para criar um service usando o Angular CLI:

  1. Na janela do terminal, navegue até o diretório que contenha sua aplicação.
  2. Coloque o seguinte comando: ng generate service nome-da-pasta-que-contem-o-modulo/nome-da-pasta/nome-do-service.

Por padrão, este comando cria o seguinte:

  • O arquivo service, nome-do-service.service.ts

Onde nome-do-service é o nome do seu component.
